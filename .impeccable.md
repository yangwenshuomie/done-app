## Design Context

### Users
Personal productivity tracker. Current: solo user. Future: public launch. Used throughout the day — quick task logging, morning planning, evening review. Mobile-only (max-width 430px).

### Brand Personality
輕盈 · 交互友好 · 简洁
Airy, interaction-friendly, iOS-native feel. Like a well-crafted Apple app: purposeful whitespace, smooth gestures, nothing unnecessary.

### Aesthetic Direction
Light mode only. Clean, minimal, restrained color. Maximum one accent color. No gradients, no decorative elements, no busy backgrounds. Feels like it belongs on iOS — not a web app pretending.

Anti-references: busy dashboards, dark hacker aesthetic, colorful productivity apps (Notion, Todoist with full color palettes), corporate SaaS.

### Design Principles
1. **Restraint first** — if an element doesn't help the user complete a task, remove it
2. **Touch-native** — every interaction should feel satisfying on a phone screen
3. **Calm hierarchy** — clear without being loud; important things surface, the rest recedes
4. **Warm neutrals** — not cold/clinical white; slightly warm off-white surfaces with tinted neutrals
5. **iOS-feel without copying** — rounded, smooth, generous padding; not skeuomorphic, not flat/Android

### Typography Notes
Current fonts DM Sans + DM Serif Display are on the reject list — replace.
CJK strategy: Western font for Latin/ASCII, system CJK fallback (`-apple-system, 'PingFang SC', sans-serif`).
Candidate: Figtree (geometric, slightly rounded terminals — iOS-adjacent, not on reject list).
